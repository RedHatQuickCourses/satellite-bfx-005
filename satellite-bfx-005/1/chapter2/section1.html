Guided Solution

Objectives

  • Investigate why the httpd service is not running on the Satellite.

  • Solve the upgrade issue in the hands-on lab environment.

Instructions

Run following instructions on satellite system with root as user.

Sample output:
[ec2-user@satellite ~]$ sudo su -
[root@satellite ~]#

  1. Try to restart the services on the Satellite

    satellite-maintain service restart
    Sample output:
    # satellite-maintain service restart
Running Restart Services
================================================================================
Check if command is run as root user:                                 [OK]
--------------------------------------------------------------------------------
Restart applicable services:

Restarting the following service(s):
redis, postgresql, pulpcore-api, pulpcore-content, pulpcore-worker@1.service, pulpcore-worker@2.service, tomcat, dynflow-sidekiq@orchestrator, foreman, httpd, dynflow-sidekiq@worker-1, dynflow-sidekiq@worker-hosts-queue-1, foreman-proxy
/ restarting httpd
Job for httpd.service failed because the control process exited with error code.
See "systemctl status httpd.service" and "journalctl -xe" for details.
- All services restarted                                              [OK]
--------------------------------------------------------------------------------

  2. /var/log/httpd/foreman-ssl_error_ssl.log shows the following message.

    AH01898: Unable to configure permitted SSL ciphers
SSL Library Error: error:1410D0B9:SSL routines:SSL_CTX_set_cipher_list:no cipher match

  3. The error indicates that the there is some issue with the TLS ciphers on the system.

  4. List the ciphers that are allowed with the following command.

    openssl ciphers -v
    Sample output:
    # satellite-maintain service restart
TLS_AES_256_GCM_SHA384  TLSv1.3 Kx=any      Au=any  Enc=AESGCM(256) Mac=AEAD
TLS_CHACHA20_POLY1305_SHA256 TLSv1.3 Kx=any      Au=any  Enc=CHACHA20/POLY1305(256) Mac=AEAD
TLS_AES_128_GCM_SHA256  TLSv1.3 Kx=any      Au=any  Enc=AESGCM(128) Mac=AEAD

  5. The command list significantly smaller number of ciphers that normal.

  6. Check the crypto policy of the system with the following command:

    update-crypto-policies --show
    Sample output:
    # update-crypto-policies --show
FIP

  7. The system is configured to use a non-standard crypto policy. Satellite servers can be configured with either DEFAULT or FIPS crypto policies. Custom policy are not supported.

  8. Switch the system back to DEFAULT crypto policy.

    update-crypto-policies --set DEFAULT
    Sample output:
    # update-crypto-policies --set DEFAULT
Setting system policy to DEFAULT
Note: System-wide crypto policies are applied on application start-up.
It is recommended to restart the system for the change of policies
to fully take place.

  9. For the policies to take full effect, reboot the system.

    reboot

  10. Once the system is available, confirm if the services are running.

    hammer ping
